# 旅行规划 Web App 设计文档

**日期**: 2026-04-30  
**状态**: 已审核  
**版本**: 1.0

---

## 一、产品定位

将模糊的旅游想法转化为清晰的旅行规划的一站式 AI 应用。用户用自然语言描述旅行期待，AI 理解意图、聚合多平台数据、生成结构化行程，并始终提供 3-5 个选项让用户自主决策。

**核心设计哲学**：用户表达意图，AI 负责结构化。不剥夺用户选择权，而是帮助用户把模糊想法变成清晰计划。

```
┌─────────────────────────────────────────────────────┐
│                  Next.js 应用                        │
│                                                     │
│  前端 (React)              后端 (API Routes)         │
│  ┌─────────────┐          ┌──────────────────────┐  │
│  │ 首页/搜索    │          │ /api/plan/generate   │  │
│  │ 行程规划页   │ ◄──────► │ /api/flights/search  │  │
│  │ 价格对比页   │          │ /api/hotels/search   │  │
│  │ 个人档案页   │          │ /api/trains/search   │  │
│  └─────────────┘          │ /api/routes          │  │
│                           └──────────────────────┘  │
└────────────────────────┬────────────────────────────┘
                         │
        ┌────────────────┼────────────────┐
        ▼                ▼                ▼
  ┌──────────┐    ┌──────────────┐  ┌──────────────┐
  │Claude API│    │  聚合数据源   │  │  MCP 工具    │
  │ (AI大脑) │    │ Amadeus 机票 │  │  12306 火车  │
  └──────────┘    │ Booking 酒店 │  │  高德 路线   │
                  │ RapidAPI     │  │  小红书 攻略 │
                  └──────────────┘  └──────────────┘
```

**数据流**：
1. 用户输入目的地 + 自然语言偏好描述
2. Claude 解析意图，拆解为结构化查询条件
3. 后端并行调用：机票 + 酒店 + 火车 + 本地攻略
4. Claude 综合所有数据 + 用户偏好 → 生成结构化行程（流式输出）
5. 用户对话式调整 → AI 只重新生成受影响部分

## 四、页面设计

### 4.1 首页 `/`

用户旅行意图的入口，以自然语言输入为核心，无预设选项限制。

**输入字段**：
- 目的地（文本）
- 出发城市（文本）
- 出发日期 + 天数（日期选择器）
- **总预算**：精确数字输入（¥ ___），含交通+住宿+餐饮
- **住宿期待**：自由文本框，例："想住在森林里的小木屋，最好有壁炉，能听到虫鸣声"
- **旅行体验期待**：自由文本框，例："想去当地人才知道的小馆子，希望有一个下午什么都不做，就坐在海边发呆"

### 4.2 行程规划页 `/plan/[id]`

双栏布局：左侧 AI 对话，右侧逐日行程。

**左栏（AI 助手）**：
- 显示 AI 生成状态和说明
- 对话框：用户用自然语言调整行程
- 示例："把第二天下午改成轻松一点的安排"、"酒店换便宜50块的"

**右栏（逐日行程）**：
- 按天展示行程卡片，可拖拽排序
- 每个条目支持：编辑、换一个、删除
- 底部：「导出规划表」按钮（生成现有 HTML 模板）

### 4.3 价格对比页 `/compare`

机票和酒店各展示 **3-5 个选项**，每个选项附一句话 pros/cons 总结。

**机票卡片格式**：
```
✈ 航班号 · 时间 · 平台 · 价格
💬 一句话：为什么适合你 / 有什么取舍
[选这个]
```

**酒店卡片格式**：
```
🏡 酒店名 · 平台 · 价格/晚 · 评分 · 距市中心距离
💬 一句话：与你的住宿期待的契合度 / 主要取舍
[选这个]
```

AI 根据用户原始描述（如"森林小木屋"）判断每个选项与期待的契合程度，写入总结。

### 4.4 个人档案页 `/profile`

保存用户偏好和历史，下次搜索自动带入：
- 常用出发城市
- 历史旅行记录
- 默认预算范围
- 住宿/体验偏好文本（可随时修改）

## 五、AI 集成设计

### 5.1 阶段一：理解 + 拆解

用户点击「规划」后，Claude 执行：

```
输入：目的地 + 预算 + 住宿描述 + 体验描述
        ↓
拆解住宿关键词：从自然语言 → 搜索条件
  "森林小木屋+壁炉" → ["forest cabin", "木屋民宿", "林间民宿"]
        ↓
预算分配：Claude 根据目的地物价自动分配
  交通 / 住宿 / 餐饮 / 景点 / 其他
        ↓
并行调用数据源 → 汇总结果给 Claude
        ↓
Claude 生成结构化行程（流式输出）
```

### 5.2 阶段二：对话式调整

用户在规划页用自然语言提出修改，Claude 只重新生成受影响部分，其余保持不变。支持跨维度调整（时间、预算、体验风格）。

### 5.3 选项呈现规则

无论机票还是酒店，AI 输出结果**永远包含 3-5 个选项**，每个选项附：
- 客观数据（价格、评分、位置）
- 一句话主观总结（与用户期待的契合度 + 主要取舍）
- 可选标注「AI 推荐」

## 七、用户核心流程

```
填写旅行意图（自然语言）
        ↓
AI 理解 + 并行拉取数据
        ↓
查看初始行程 + 3-5个机票/酒店选项
        ↓
对话式微调（"改成慢一点的节奏"）
        ↓
确认选择 → 导出 HTML 规划表
```
